JavaScript Datatypes

In JavaScript, datatypes define the type of values we can work with. They are divided into two main categories: Primitive (call by value) and Non-Primitive (call by reference). Let’s explore each type in detail with examples.

Primitive Datatypes (Call by Value)

Primitive datatypes are stored and copied by value. This means when you assign a primitive value to another variable, JavaScript copies the actual value instead of the memory reference.

There are 7 primitive datatypes in JavaScript:

1. String

A String is a sequence of characters used to represent text. Strings are written inside quotes (single, double, or backticks) and are immutable(once you create a string, you cannot change its characters directly.). They provide many built-in methods such as toUpperCase(), slice(), and includes().
typeof: Always returns "string". 

let name = "Sajid";
console.log(typeof name);        // "string"
console.log(name.toUpperCase()); // "SAJID"
console.log(name.includes("ji")); // true

2. Number

A Number in JavaScript represents both integers and floating-point (decimal) values. JavaScript does not have separate types for integers and floats — everything numeric is stored as Number. Numbers are also immutable, which means once a number is created, it cannot be directly changed. Mathematical operations always produce a new value instead of modifying the original.
typeof: Always returns "number" (except for BigInt). 

let a = 10;
let b = a; 
b = b + 5;      

console.log(a); // 10  (unchanged, since numbers are immutable)
console.log(b); // 15  (new value created)

console.log(typeof a); // "number"
console.log(1 / 0);    // Infinity
console.log("abc" * 2); // NaN (Not-a-Number)

3. Boolean

A Boolean represents one of two values: true or false. It is commonly used in conditional statements and logical operations. Booleans are immutable, meaning once a boolean value is created, it cannot be changed. Any operation on booleans results in a new value instead of modifying the existing one.
typeof: Always returns "boolean". 

let isAdult = true;
let canVote = false;

console.log(typeof isAdult);  // "boolean"
console.log(5 > 3);           // true
console.log(10 === 5);        // false

4. Null

Null is a special value that represents intentional emptiness or "no value". It is usually assigned explicitly when you want to clear or reset a variable. Being a primitive, null is immutable — you cannot change it directly.
⚠️ However, due to a long-standing bug in JavaScript, typeof null incorrectly returns "object".
typeof: Returns "object". 

let car = null;

console.log(car);        // null
console.log(typeof car); // "object"  (quirk in JavaScript)

5. Undefined

Undefined is a special datatype that indicates a variable has been declared but no value has been assigned yet. It is also returned by functions that do not explicitly return a value. Being a primitive, undefined is immutable.
typeof: Returns "undefined". 

let x;
console.log(x);           // undefined

function greet() {}
console.log(greet());     // undefined

console.log(typeof x);    // "undefined"
console.log(typeof greet()); // "undefined"

function hello() {}
console.log(hello()); // undefined

6. Symbol

Symbol is a unique and immutable primitive value introduced in ES6. Even if two symbols have the same description, they are always different. Symbols are often used as object keys to avoid property name conflicts and to create hidden properties that don’t show up in normal object iteration.
typeof: Returns "symbol". 

let id1 = Symbol("id");
let id2 = Symbol("id");

console.log(id1 === id2); // false (always unique)

// Using Symbol as an object key
let user = {
  name: "Sajid",
  age: 20,
  [id1]: 123
};

console.log(user.name);     // Sajid
console.log(user[id1]);     // 123
console.log(Object.keys(user)); // ["name", "age"] (symbol key is hidden)

console.log(typeof id1);    // "symbol"

7. BigInt

BigInt is a special numeric datatype introduced in ES2020. It is used to represent integers that are larger than the safe limit of the Number type (which is 2^53 - 1). BigInt is created by appending n at the end of an integer literal. Unlike regular numbers, BigInt can handle very large integers without losing precision.
typeof: Returns "bigint". 

let big = 123456789012345678901234567890n;
console.log(big);       // 123456789012345678901234567890n

console.log(big + 10n); // 123456789012345678901234567900n

// Mixing BigInt with Number throws an error
// console.log(big + 5); ❌ TypeError

console.log(typeof big); // "bigint"

Non-Primitive Datatypes (Call by Reference)

Non-primitive datatypes are stored and copied by reference. This means when you assign one variable to another, both variables point to the same memory location. Changing one affects the other.

There are 3 non-primitive datatypes in JavaScript:

9. Array

An Array is a special type of object that stores an ordered collection of values. Each value is called an element, and arrays are zero-indexed, meaning the first element is at index 0. Arrays can contain elements of any datatype, including other arrays or objects. JavaScript provides many built-in methods like push(), pop(), map(), filter(), and forEach() for manipulating arrays.
typeof: Returns "object" (because arrays are implemented as objects in JavaScript). 

let fruits = ["apple", "banana", "mango"];

console.log(fruits[0]);      // apple
fruits.push("orange");       // adds "orange" at the end
console.log(fruits.length);  // 4
console.log(typeof fruits);  // "object"

8. Object

An Object is a collection of key-value pairs. Keys are usually strings (or symbols), and values can be of any datatype, including numbers, strings, booleans, functions, or even other objects. Objects are the foundation of JavaScript and are widely used to represent real-world entities.
typeof: Returns "object". 

let user = {
  name: "Sajid",
  age: 20,
  isAdmin: true
};

console.log(user.name);    // Sajid
console.log(user["age"]);  // 20
console.log(typeof user);  // "object"

10. Function

A Function is a reusable block of code designed to perform a specific task. Functions can accept inputs called parameters, process them, and return an output using the return keyword. They help in reducing repetition, improving modularity, and making code more organized. JavaScript supports different types of functions such as declaration functions, expression functions, and arrow functions.
typeof: Returns "function" (special case in JavaScript — technically functions are objects, but typeof gives "function" for clarity). 

function add(a, b) {
  return a + b;
}

console.log(add(5, 3));   // 8
console.log(typeof add);  // "function"
