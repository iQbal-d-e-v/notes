Git & GitHub – Todo App Walkthrough

Imagine you’re a developer starting a project called Todo App. You’ll use Git locally to track changes, then push it to GitHub to collaborate with a friend. This guide walks you step by step.

1. Initial Setup (telling Git who you are)

Before anything, Git needs your identity for commit history. 

git config --global user.name "Sajid"
git config --global user.email "sajid@example.com"

💡 Every commit you make will say “Sajid wrote this”. If you skip this, commits look anonymous.

2. Starting a New Project

mkdir todo-app
cd todo-app
git init

👉 git init creates a hidden folder .git/. That’s Git’s “brain” — it records every change.

3. First File & First Commit

echo "# Todo App" > README.md
git status
git add README.md
git commit -m "Initial commit: add README"

💡 git add → stage changes. 💡 git commit → save snapshot permanently with a message.

4. Editing & Tracking Changes

echo "This is a simple todo app." >> README.md
git status
git diff
git add README.md
git commit -m "docs: update project description"

👉 git diff shows what changed line by line.

5. Branching (work without breaking main code)

git branch login-feature
git switch login-feature
echo "Login form code here" > login.js
git add login.js
git commit -m "feat: add login feature"

Switch back with git switch main. Notice login.js disappears because it only exists in that branch.

6. Merging Branches

git switch main
git merge login-feature

✅ If no conflicts → Git merges automatically. ⚠️ If conflicts → Git marks the file, you fix it, then: 

git add fixed-file.js
git commit

7. Connecting to GitHub

On GitHub → create New Repository → call it todo-app. 

git remote add origin git@github.com:yourname/todo-app.git
git push -u origin main

💡 Now your project exists both locally and on GitHub.

8. Collaboration (friend helps)

git clone git@github.com:yourname/todo-app.git
git switch -c task-list
# edit files
git add .
git commit -m "feat: add task list"
git push -u origin task-list

On GitHub, your friend opens a Pull Request → you review & merge.

9. Keeping Up-to-date

git pull     # fetch + merge new stuff
git fetch    # only see what’s new

10. Fixing Mistakes

git restore --staged file.txt       # unstage
git reset --soft HEAD~1             # undo last commit, keep changes
git reset --hard HEAD~1             # undo & delete changes
git reflog                          # recover lost commits

11. Real Use Case Flow (Summary)

git init
git status
git add .
git commit -m "message"
git branch feature
git switch feature
git merge feature
git push
git pull
git restore / git reset / git reflog

✅ Git = your local time machine & branch manager. ✅ GitHub = your team’s online hub for sharing, reviewing & collaborating.
