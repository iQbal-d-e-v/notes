JavaScript Strings & String Methods

Deep explanation with examples, compiled in a clean and minimal website format.

1. What is a String in JavaScript?

1.1 Definition of String

A string in JavaScript is a sequence of characters used to represent text. Strings are enclosed in quotes, and they can contain letters, numbers, symbols, or spaces.

// Example of string declaration
let singleQuoteString = 'Hello';    // Using single quotes
let doubleQuoteString = "World";    // Using double quotes
let templateLiteralString = `Hi there`; // Using backticks (template literal)

console.log(singleQuoteString);  // Output: Hello
console.log(doubleQuoteString);  // Output: World
console.log(templateLiteralString);  // Output: Hi there

1.2 Immutability of Strings

Strings in JavaScript are immutable, meaning once a string is created, you cannot change its individual characters directly. Instead, string methods return a new string.

// Trying to change a character does not work
let str = "Hello";
str[0] = "J";  // This does NOT modify the string
console.log(str);  // Output: "Hello"

// Instead, create a new string
let newStr = "J" + str.slice(1);
console.log(newStr);  // Output: "Jello"

1.3 Types of Quotes

You can define strings using three types of quotes in JavaScript:

  • Single Quotes: 'string'
  • Double Quotes: "string"
  • Backticks (Template Literals): `string`, which allow embedding expressions. 
// Single and double quotes are equivalent
let single = 'Hello';
let double = "World";

console.log(single + " " + double);  // Output: Hello World

// Template literals allow expression embedding
let name = "Sajid";
let greeting = `Hello, ${name}!`;
console.log(greeting);  // Output: Hello, Sajid!

1.4 Template Literals vs Regular Strings

Template literals (using backticks `...`) are more powerful than regular strings. They allow for expression interpolation and multi-line strings.

let a = 10;
let b = 20;

// Using template literal with embedded expressions
let result = `The sum of ${a} and ${b} is ${a + b}`;
console.log(result);  // Output: The sum of 10 and 20 is 30

// Multi-line string using template literal
let multiLine = `Line 1
Line 2
Line 3`;

console.log(multiLine);
/*
Output:
Line 1
Line 2
Line 3
*/

1.5 Common Use Cases of Strings

Strings are essential for many tasks in JavaScript, such as:

  • Displaying messages in the console or on a webpage.
  • Collecting user input and processing it.
  • Manipulating text for data formatting, reports, or logs.
// Example: Concatenating user input
let firstName = "John";
let lastName = "Doe";

let fullName = firstName + " " + lastName;
console.log("User Full Name:", fullName);  // Output: User Full Name: John Doe

2. How Strings are Stored

In JavaScript, strings are stored as sequences of UTF-16 code units. They are immutable and provide indexed access to individual characters, similar to arrays, but without array methods. Understanding their storage helps with efficient string manipulation and avoiding common pitfalls.

2.1 Array-like Character Access

Strings allow accessing individual characters by their index position, starting from 0. This behavior makes strings appear like arrays for reading purposes, but they do not inherit array methods.

const word = "JavaScript";

// Access characters by index
console.log(word[0]);  // Output: "J"
console.log(word[4]);  // Output: "S"

// Confirm string type
console.log(typeof word);         // Output: "string"
console.log(Array.isArray(word)); // Output: false

2.2 String Immutability

Strings in JavaScript cannot be changed after they are created. Any operation that appears to modify a string actually produces a new string, leaving the original unchanged.

const originalText = "Immutable";

// Attempt to change character directly (will not work)
originalText[0] = "M";
console.log(originalText);  // Output: "Immutable"

// Correct approach: Create a new string
const updatedText = "M" + originalText.slice(1);
console.log(updatedText);  // Output: "Mutable"

2.3 Looping Through Characters

You can loop through strings using a standard for loop to process or analyze each character based on its index.

const sample = "Loop";

// Iterate through each character
for (let i = 0; i < sample.length; i++) {
  console.log(`Character at index ${i}: ${sample[i]}`);
}

/* Output:
Character at index 0: L
Character at index 1: o
Character at index 2: o
Character at index 3: p
*/

3. String Methods in JavaScript

JavaScript provides various built-in methods and properties to manipulate and work with strings. Each method serves a unique purpose such as transforming, searching, extracting, or formatting string values effectively while respecting immutability.

3.1 length Property

The .length property returns the total number of characters in a string, including spaces, numbers, and special characters. It is commonly used for validation or to control loops when processing strings. 

const text = "Hello World";

console.log(text.length);  // Output: 11

const emptyText = "";
console.log(emptyText.length);  // Output: 0

3.2 toUpperCase()

.toUpperCase() converts all characters of a string to uppercase and returns a new string.

const msg = "JavaScript";

const upper = msg.toUpperCase();
console.log(upper);  // Output: "JAVASCRIPT"

3.3 toLowerCase()

.toLowerCase() converts all characters of a string to lowercase and returns a new string.

const msg = "JavaScript";

const lower = msg.toLowerCase();
console.log(lower);  // Output: "javascript"

3.4 indexOf()

.indexOf() returns the index of the first occurrence of a specified substring, or -1 if not found. 

const sentence = "I love JavaScript";

const index = sentence.indexOf("love");
console.log(index);  // Output: 2

3.5 includes()

.includes() checks whether a substring exists in the string and returns true or false.

const sentence = "I love JavaScript";

const hasJava = sentence.includes("Java");
console.log(hasJava);  // Output: true

3.6 startsWith()

.startsWith() checks if the string starts with the given substring and returns true or false.

const sentence = "I love JavaScript";

const startsWithI = sentence.startsWith("I");
console.log(startsWithI);  // Output: true

3.7 endsWith()

.endsWith() checks if the string ends with the given substring and returns true or false.

const sentence = "I love JavaScript";

const endsWithPt = sentence.endsWith("pt");
console.log(endsWithPt);  // Output: true

3.8 slice()

.slice() extracts a section of a string and returns it as a new string.

const str = "Hello World";

const part1 = str.slice(0, 5);
console.log(part1);  // Output: "Hello"

3.9 substring()

.substring() returns a part of the string between two indices.

const str = "Hello World";

const part2 = str.substring(6, 11);
console.log(part2);  // Output: "World"

3.10 substr()

.substr() returns a substring starting at a specified index for a given length.

const str = "Hello World";

const part3 = str.substr(6, 5);
console.log(part3);  // Output: "World"

3.11 replace()

.replace() replaces the first occurrence of a substring with a new value.

const text = "I love JavaScript. JavaScript is fun.";

const replacedOnce = text.replace("JavaScript", "JS");
console.log(replacedOnce);  
// Output: "I love JS. JavaScript is fun."

3.12 replaceAll()

.replaceAll() replaces all occurrences of a substring with a new value.

const text = "I love JavaScript. JavaScript is fun.";

const replacedAll = text.replaceAll("JavaScript", "JS");
console.log(replacedAll);  
// Output: "I love JS. JS is fun."

3.13 trim()

.trim() removes spaces from both the beginning and end of a string.

const messy = "   Hello JS   ";

console.log(messy.trim());  // Output: "Hello JS"

3.14 trimStart()

.trimStart() removes spaces only from the beginning of a string.

const messy = "   Hello JS   ";

console.log(messy.trimStart());  // Output: "Hello JS   "

3.15 trimEnd()

.trimEnd() removes spaces only from the end of a string.

const messy = "   Hello JS   ";

console.log(messy.trimEnd());  // Output: "   Hello JS"

3.16 split()

.split() divides a string into an array based on a separator.

const data = "apple,banana,grape";

const fruitsArray = data.split(",");
console.log(fruitsArray);  
// Output: ["apple", "banana", "grape"]

3.17 join()

.join() combines elements of an array into a string using a separator.

const fruitsArray = ["apple", "banana", "grape"];

const joinedFruits = fruitsArray.join(" | ");
console.log(joinedFruits);  
// Output: "apple | banana | grape"

3.18 repeat()

.repeat() repeats a string a specified number of times.

const greeting = "Hi! ";

const repeated = greeting.repeat(3);
console.log(repeated);  
// Output: "Hi! Hi! Hi! "

3.19 charAt()

.charAt(index) returns the character at a specified index.

const s = "ABC";

console.log(s.charAt(1));  // Output: "B"

3.20 charCodeAt()

.charCodeAt(index) returns the UTF-16 code of the character at the specified index.

const s = "ABC";

console.log(s.charCodeAt(1));  // Output: 66

3.21 concat()

.concat() concatenates multiple strings into a new string.

const a = "Hello";
const b = "World";

console.log(a.concat(" ", b));  // Output: "Hello World"

3.22 padStart()

.padStart() pads the string at the beginning to reach a specified length.

const num = "5";

console.log(num.padStart(3, "0"));  // Output: "005"

3.23 padEnd()

.padEnd() pads the string at the end to reach a specified length.

const num = "5";

console.log(num.padEnd(3, "*"));  // Output: "5**"

3.24 Template Literals

Template literals allow embedding expressions and multi-line strings with backticks, improving readability over traditional concatenation.

const name = "Sajid";
const age = 21;

const message = `My name is ${name} and I am ${age} years old.`;
console.log(message);  
// Output: "My name is Sajid and I am 21 years old."

4. Summary of Key String Concepts

This section summarizes the essential points about JavaScript strings and their methods to reinforce your understanding and help you quickly recall the main ideas when writing code.

4.1 Strings are Immutable

Once a string is created, its characters cannot be changed directly. Any modification creates a new string by combining parts of the original string.

4.2 Use Template Literals for Readability

Template literals using backticks allow embedding variables and expressions directly into strings, which makes code more readable compared to using concatenation with the + operator.

4.3 slice() vs substring()

.slice() supports negative indices to extract characters from the end of a string, while .substring() does not support negative indices. This makes .slice() more flexible for certain use cases.

4.4 includes() vs indexOf()

.includes() returns a simple boolean indicating if a substring exists, improving readability. .indexOf() returns the index of the substring or -1, which requires a separate comparison to check existence.

4.5 Always Trim Input

.trim() is important to remove extra spaces from user input before saving or processing it, preventing bugs and ensuring data consistency.
