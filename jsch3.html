1. What is a String in JavaScript?

  • A string is a sequence of characters enclosed in quotes.
  • Strings are immutable → once created, you can’t change characters inside it; you create a new string.
  • You can use 'single', "double", or `backticks` (template literals).
let str1 = "Hello";    // double quotes
let str2 = 'World';    // single quotes
let str3 = `Hello ${str2}`; // template literal
console.log(str3); // Hello World

2. How Strings are Stored

  • A string is array-like (you can access characters by index), but not an actual array.
  • Indices start at 0.
let word = "JavaScript";
console.log(word[0]);  // J
console.log(word[4]);  // S
console.log(word.length); // 10

3. String Methods in JavaScript

JavaScript provides built-in methods to manipulate strings. Here’s a detailed explanation of the most important ones:

3.1 length (Property, not method)

Gives the number of characters.

let txt = "Hello World";
console.log(txt.length); // 11

3.2 Case Conversion

  • .toUpperCase() → Converts to UPPERCASE.
  • .toLowerCase() → Converts to lowercase.
let msg = "JavaScript";
console.log(msg.toUpperCase()); // "JAVASCRIPT"
console.log(msg.toLowerCase()); // "javascript"

3.3 Searching Methods

  • .indexOf(substring) → First index or -1.
  • .lastIndexOf(substring) → Last occurrence.
  • .includes(substring) → Boolean contains check.
  • .startsWith(substring) → Checks beginning.
  • .endsWith(substring) → Checks ending.
let sentence = "I love JavaScript";
console.log(sentence.indexOf("love"));  // 2
console.log(sentence.includes("Java")); // true
console.log(sentence.startsWith("I"));  // true
console.log(sentence.endsWith("pt"));   // true

3.4 Extracting Part of a String

  • .slice(start, end) → Extracts part (end excluded). Supports negatives.
  • .substring(start, end) → Similar to slice but no negative index.
  • .substr(start, length) → deprecated but still works.
let str = "Hello World";
console.log(str.slice(0, 5));      // "Hello"
console.log(str.substring(6, 11)); // "World"
console.log(str.substr(6, 5));     // "World"

3.5 Replacing

  • .replace("old", "new") → Replaces first match.
  • .replaceAll("old", "new") → Replaces all matches (ES2021).
let text = "I love JavaScript. JavaScript is fun.";
console.log(text.replace("JavaScript", "JS"));  
// "I love JS. JavaScript is fun."

console.log(text.replaceAll("JavaScript", "JS"));  
// "I love JS. JS is fun."

3.6 Removing Extra Spaces

  • .trim() → Removes spaces from both ends.
  • .trimStart() → Removes spaces at start.
  • .trimEnd() → Removes spaces at end.
let messy = "   Hello JS   ";
console.log(messy.trim());      // "Hello JS"
console.log(messy.trimStart()); // "Hello JS   "
console.log(messy.trimEnd());   // "   Hello JS"

3.7 Splitting & Joining

  • .split(separator) → Splits string into array.
  • .join(separator) → (array method) Useful after split.
let data = "apple,banana,grape";
let arr = data.split(",");
console.log(arr); // ["apple", "banana", "grape"]

3.8 Repeating

.repeat(n) → Repeats string n times.

console.log("Hi!".repeat(3)); // "Hi!Hi!Hi!"

3.9 Character Access

  • .charAt(index) → Get character.
  • .charCodeAt(index) → Unicode value.
  • [index] → Alternative access.
let s = "ABC";
console.log(s.charAt(1));     // "B"
console.log(s.charCodeAt(1)); // 66 (Unicode for B)

3.10 Concatenation

  • .concat() → Joins strings.
  • + operator (common in practice).
  • Template literals (best for readability).
let a = "Hello";
let b = "World";
console.log(a.concat(" ", b));   // "Hello World"
console.log(a + " " + b);        // "Hello World"
console.log(`${a} ${b}`);        // "Hello World"

3.11 Padding (formatting)

  • .padStart(length, char)
  • .padEnd(length, char)
let num = "5";
console.log(num.padStart(3, "0")); // "005"
console.log(num.padEnd(3, "*"));   // "5**"

3.12 Template Literals

  • Use `backticks` for multi-line strings and embedding variables.
let name = "Sajid";
let age = 21;
console.log(`My name is ${name} and I am ${age} years old.`);
// My name is Sajid and I am 21 years old.

4. Key Things to Remember

  1. Strings are immutable.
  2. Use template literals for readability.
  3. slice is more powerful than substring (allows negative indices).
  4. For searching, includes is simpler than indexOf.
  5. Use .trim() before saving user input.

Why not mutate a string directly?

Because strings are immutable in JavaScript. When you "change" a string, engines actually create a new string and return it. For example, str[0] = 'X' does nothing; use methods (like slice, replace) to build a new string instead.
