3. Representation of Data

Once the data is collected, the next step is to represent it clearly so we can understand patterns.

(a) For Small Data

We can use simple charts and tables:

  • Frequency table: Shows how often each value occurs.
  • Frequency polygon: A graph that shows frequencies with connected points.
  • Pie diagram (pie chart): Shows proportions using slices of a circle.

(b) For Large Data

We need stronger methods because the dataset is huge:

  • Histogram: A bar-like graph showing how data is spread over intervals.
  • Ogives (cumulative distribution graphs): Help us see running totals and percentiles.
  • Stem-and-leaf diagram: Shows actual data values in a structured form, useful for identifying patterns.

4. Summarizing Data

Instead of just looking at tables and graphs, we often want a single number to describe data.

These are called measures of central tendency:

  • Mean (average): The balance point of the data.
  • Median: The middle value when the data is arranged.
  • Mode: The most frequent value.

But numbers also vary, so we need measures of variability:

  • Variance: How far data values spread out from the mean.
  • Standard deviation: A square-root form of variance, easier to interpret.
  • Sample variance/standard deviation: Same as above but calculated for a sample.

ðŸ‘‰ These measures help us understand how consistent or spread out the data is.

6. Normal Distribution

One very common data pattern in the real world is the normal distribution (bell-shaped curve).

Examples: height of people, exam scores, weight of newborn babies.

  • Most data lies near the mean (center).
  • Very few data points lie far away (extreme values).

Understanding normal distribution is essential because many statistical methods are based on it.
