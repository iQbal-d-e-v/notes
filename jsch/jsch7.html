JavaScript Arrays & Methods (In-Depth)

Arrays are one of the most important data structures in JavaScript. They allow us to store multiple values in a single variable, and JavaScript provides a rich set of built-in methods for working with them. In this page, we‚Äôll go step by step: from the basics of arrays, to every method with code examples, and best practices.

1. What is an Array?

An array is a list-like object used to store multiple values in a single variable. Arrays are zero-indexed, meaning the first element is at position 0. They can store mixed data types (numbers, strings, objects, even other arrays).

let fruits = ["Apple", "Banana", "Mango"]; console.log(fruits[0]); // "Apple" console.log(fruits.length); // 3

- Arrays in JavaScript are actually special objects. - The length property dynamically updates when you add or remove elements. - Arrays are stored in contiguous memory internally, optimized for fast lookups by index.

2. Basic Methods

push()

Adds an element to the end of the array.

let arr = [1, 2, 3]; arr.push(4); console.log(arr); // [1, 2, 3, 4]

Complexity: O(1) usually.

pop()

Removes the last element from the array.

let arr = [1, 2, 3]; arr.pop(); console.log(arr); // [1, 2]

shift()

Removes the first element (shifts all others).

let arr = [10, 20, 30]; arr.shift(); console.log(arr); // [20, 30]

Note: Slower than pop() since all elements shift ‚Üí O(n).

unshift()

Adds elements to the start of the array.

let arr = [20, 30]; arr.unshift(10); console.log(arr); // [10, 20, 30]

3. Iteration Methods

forEach()

Executes a function once for each array element (no return value).

[1, 2, 3].forEach(num => console.log(num * 2)); // Output: 2, 4, 6

map()

The map() method in JavaScript is used to transform each element of an array and return a new array with the results. Importantly, map() does not change (mutate) the original array‚Äîit always returns a fresh one.

üìå Syntax:

array.map(function(currentValue, index, array) { // return new value for the new array }, thisArg);
  • currentValue ‚Üí the element currently being processed.
  • index ‚Üí (optional) index of the element.
  • array ‚Üí (optional) the original array on which map() was called.
  • thisArg ‚Üí (optional) value to use as this inside the callback.

‚úÖ Example 1: Basic Transformation

{ let numbers = [1, 2, 3, 4]; let doubled = numbers.map(num => num * 2); console.log(doubled); // [2, 4, 6, 8] console.log(numbers); // [1, 2, 3, 4] (unchanged) }

Each element is doubled, and a new array is returned. Notice that the original numbers array is not modified.

‚úÖ Example 2: Using index parameter

{ let fruits = ["Apple", "Banana", "Cherry"]; let labeled = fruits.map((fruit, index) => `${index + 1}. ${fruit}`); console.log(labeled); // ["1. Apple", "2. Banana", "3. Cherry"] }

Here we used the index parameter to add numbering to each fruit.

‚úÖ Example 3: Working with Objects

{ let users = [ {name: "Alice", age: 20}, {name: "Bob", age: 25}, {name: "Charlie", age: 30} ]; let names = users.map(user => user.name); console.log(names); // ["Alice", "Bob", "Charlie"] }

A very common use of map() is extracting a single property from an array of objects.

‚úÖ Example 4: Chaining map()

{ let nums = [1, 2, 3, 4]; let result = nums .map(x => x * 2) // double each number .map(x => x + 1); // then add 1 console.log(result); // [3, 5, 7, 9] }

Since map() always returns a new array, you can chain multiple map() calls.

‚ö†Ô∏è Common Mistakes

  • Forgetting return: In regular functions, you must return a value. Without return, the new array will contain undefined.
  • Expecting mutation: map() does not change the original array. Use forEach() if you want side effects instead.

üí° Real-World Use Cases

  • Converting an array of numbers (e.g., Celsius ‚Üí Fahrenheit).
  • Extracting fields from database results (e.g., users.map(u => u.id)).
  • Generating HTML elements from data (in React/JS frameworks).

filter()

Returns a new array with elements that pass a condition.

let nums = [5, 12, 8, 130, 44]; let big = nums.filter(x => x > 10); console.log(big); // [12, 130, 44]

reduce()

Reduces the array to a single value.

let nums = [1, 2, 3, 4]; let sum = nums.reduce((acc, val) => acc + val, 0); console.log(sum); // 10

some()

Checks if at least one element satisfies a condition.

let nums = [1, 2, 3]; console.log(nums.some(x => x > 2)); // true

every()

Checks if all elements satisfy a condition.

let nums = [2, 4, 6]; console.log(nums.every(x => x % 2 === 0)); // true

4. Searching & Index Methods

find()

Returns the first element that matches a condition.

let arr = [10, 20, 30]; console.log(arr.find(x => x > 15)); // 20

findIndex()

Returns the index of the first matching element.

let arr = [10, 20, 30]; console.log(arr.findIndex(x => x > 15)); // 1

indexOf()

Finds the index of a value.

let arr = ["a", "b", "c"]; console.log(arr.indexOf("b")); // 1

lastIndexOf()

Finds the last occurrence of a value.

let arr = ["a", "b", "c", "b"]; console.log(arr.lastIndexOf("b")); // 3

includes()

Checks if an array contains a value.

let arr = [1, 2, 3]; console.log(arr.includes(2)); // true

5. Modification Methods

slice()

Returns a shallow copy of a portion of an array.

let arr = [1, 2, 3, 4, 5]; console.log(arr.slice(1, 3)); // [2, 3]

splice()

Adds/removes elements in place.

let arr = [1, 2, 3, 4]; arr.splice(1, 2, 99); console.log(arr); // [1, 99, 4]

concat()

Merges arrays into a new one.

let arr1 = [1, 2]; let arr2 = [3, 4]; console.log(arr1.concat(arr2)); // [1, 2, 3, 4]

join()

Converts array into string with separator.

let arr = ["Hello", "World"]; console.log(arr.join(" ")); // "Hello World"

6. Sorting & Reversing

sort()

Sorts the array (by default lexicographic).

let arr = [10, 2, 5]; arr.sort(); console.log(arr); // [10, 2, 5] ‚Üí "wrong" lexicographic arr.sort((a, b) => a - b); console.log(arr); // [2, 5, 10]

reverse()

Reverses the array in place.

let arr = [1, 2, 3]; arr.reverse(); console.log(arr); // [3, 2, 1]

7. Flattening & Advanced

flat()

Flattens nested arrays.

let arr = [1, [2, [3, 4]]]; console.log(arr.flat(2)); // [1, 2, 3, 4]

flatMap()

Maps + flattens by one level.

let arr = [1, 2, 3]; console.log(arr.flatMap(x => [x, x*2])); // [1, 2, 2, 4, 3, 6]

8. Summary & Best Practices

  • Use map, filter, reduce for clean functional style.
  • Use slice for copies instead of splice unless you need mutation.
  • Be careful with sort() since default is string-based.
  • push/pop are faster than shift/unshift.
