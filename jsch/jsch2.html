JavaScript Operators

In this page, we are going to learn about different types of operators in JavaScript, how they work, and see examples for each one. Operators are special symbols used to perform operations on values and variables.

1. Arithmetic Operators

Arithmetic operators perform mathematical operations on values.
They usually work on numbers, but JavaScript automatically converts other types (like strings) to numbers in many cases.

Addition (+)

The + operator is used to add numbers.
It also concatenates strings if any operand is a string. 


// Number addition
let a = 10, b = 3;
console.log(a + b); // 13 # adds normally

// String concatenation
console.log("10" + b); // "103" # b converted to string → reason: 
                      //if any operand is string, + concatenates

// Concatenating 2 strings
console.log("Hello, " + "World!"); // "Hello, World!" # explicit string concatenation

# Additional info: + is the only arithmetic operator that can concatenate strings.

Subtraction (-)

Subtracts the right-hand value from the left-hand value.
Non-number strings are converted to numbers if possible. 


console.log(a - b);    // 7 # normal subtraction
console.log("10" - 3); // 7 # "10" converted to number → reason: - always converts to number
console.log("hello" - 3); // NaN # cannot convert "hello" → reason: invalid numeric conversion

Multiplication (*)

Multiplies two numbers.
Strings that represent numbers are automatically converted.
Invalid strings give NaN. 


console.log(a * b);    // 30 # normal multiplication
console.log("5" * 2);  // 10 # "5" converted to number
console.log("hello" * 2); // NaN # invalid string → reason: cannot multiply non-numeric string

Division (/)

Divides the left operand by the right.
Division by zero gives Infinity or -Infinity.
Strings are converted if numeric. 


console.log(a / b);    // 3.3333333333333335 # normal division
console.log(10 / 0);   // Infinity # division by zero → JS follows IEEE 754
console.log(-10 / 0);  // -Infinity # negative divided by zero
console.log("10" / "2"); // 5 # strings converted to numbers
console.log("hello" / 2); // NaN # invalid string

Modulus (%)

Returns the remainder after dividing the left operand by the right.
Useful for checking divisibility or cycles. 


console.log(a % b);  // 1 # remainder of 10 / 3
console.log(10 % 2); // 0 # divisible exactly
console.log(7 % 4);  // 3 # 7 divided by 4 leaves remainder 3

// Negative numbers
console.log(-7 % 3); // -1 # sign follows the dividend → reason: JS keeps left operand sign

Exponentiation (**)

Raises the left operand to the power of the right operand.
Equivalent to Math.pow(). 


console.log(a ** b);    // 1000 # 10^3
console.log(2 ** 4);    // 16 # 2*2*2*2
console.log(9 ** 0.5);  // 3 # square root → exponent 0.5
console.log(5 ** -1);   // 0.2 # negative exponent → 1/(5^1)

# Additional info: ** has higher precedence than * / % → reason: evaluated before other arithmetic operations

2. Assignment Operators

Assignment operators are used to assign values to variables.
They can also perform an operation and assign the result in one step.
The simplest assignment is =, but others like +=, -=, *=, /=, %=` are called **compound assignment operators**.

Simple Assignment (=)

Assigns the value on the right-hand side to the variable on the left-hand side. 


let x = 10; // x is assigned 10
console.log(x); // 10 # simple assignment

# Important: Always declare variables with let or const to avoid creating global variables → reason: avoids unintended side effects

Addition Assignment (+=)

Adds the right-hand value to the variable and assigns the result to the variable.
If the variable is a string, it concatenates instead of adding numerically. 


x += 5; // x = x + 5 → 15
console.log(x); // 15 # numeric addition

let str = "Hello, ";
str += "World!";
console.log(str); // "Hello, World!" # string concatenation → 
                  //reason: += behaves like + for strings

Subtraction Assignment (-=)

Subtracts the right-hand value from the variable and assigns the result to it.
Strings are converted to numbers if possible. 


x -= 3; // x = x - 3 → 12
console.log(x); // 12 # numeric subtraction

let y = "20";
y -= 5;
console.log(y); // 15 # string converted to number → reason: - always converts to number

Multiplication Assignment (*=)

Multiplies the variable by the right-hand value and assigns the result.
Strings representing numbers are converted automatically. 


x *= 2; // x = x * 2 → 24
console.log(x); // 24 # numeric multiplication

let z = "5";
z *= 3;
console.log(z); // 15 # string converted to number → reason: * always converts to number

Division Assignment (/=)

Divides the variable by the right-hand value and assigns the result.
Division by zero returns Infinity or -Infinity. Strings converted if numeric. 


x /= 4; // x = x / 4 → 6
console.log(x); // 6

let a = "20";
a /= 5;
console.log(a); // 4 # string converted to number

let b = 10;
b /= 0;
console.log(b); // Infinity # division by zero → JS follows IEEE 754

Modulus Assignment (%=)

Takes the remainder after dividing the variable by the right-hand value and assigns it.
Useful for cycles, like looping numbers. 


x %= 5; // x = x % 5 → 1
console.log(x); // 1

let n = 14;
n %= 4;
console.log(n); // 2 # 14 divided by 4 leaves remainder 2

# Additional info: Compound assignment operators are shorthand for performing an operation and assigning the result at the same time → reason: makes code shorter and easier to read

3. Comparison Operators

Comparison operators are used to compare two values.
They return true or false based on the comparison.
They are often used in conditions for if-statements, loops, and logical operations.

Equal to (==)

Checks if the values are equal, but ignores the type.
# Important: Type coercion occurs, so a string containing a number can be equal to a number. 


console.log(5 == "5"); // true
/*
  # reason: "5" converted to number 5 → values equal
*/
console.log(0 == false); // true
/*
  # reason: false converted to 0 → values equal
*/

Strict Equal to (===)

Checks if both value and type are equal.
# Important: No type coercion occurs, safer to use for precise comparison. 


console.log(5 === "5"); // false
/*
  # reason: number 5 is not same type as string "5"
*/
console.log(10 === 10); // true
/*
  # reason: value and type both match
*/

Not Equal (!=)

Checks if values are not equal, ignoring type.
# Important: Type coercion occurs similar to ==. 


console.log(7 != 8); // true
/*
  # reason: 7 and 8 are different values
*/
console.log(5 != "5"); // false
/*
  # reason: "5" converted to number → values equal
*/

Strict Not Equal (!==)

Checks if value or type is not equal.
# Important: No type coercion, only true if value or type differ. 


console.log(7 !== "7"); // true
/*
  # reason: types differ → number vs string
*/
console.log(10 !== 10); // false
/*
  # reason: value and type are same
*/

Greater than (>) and Less than (<)

Compares numeric or string values.
# Important: Strings are compared lexicographically (Unicode order). 


console.log(10 > 6); // true
/*
  # reason: 10 is greater than 6
*/
console.log(4 < 2); // false
/*
  # reason: 4 is not less than 2
*/
console.log("apple" > "banana"); // false
/*
  # reason: "a" comes before "b" in Unicode → apple < banana
*/

Greater than or equal to (>=) and Less than or equal to (<=)

Checks for greater/less or equality.
# Important: Includes equality in comparison. 


console.log(5 >= 5); // true
/*
  # reason: value equals → >= includes equality
*/
console.log(3 <= 2); // false
/*
  # reason: 3 is greater than 2 → <= checks less than or equal
*/

# Additional info: Always prefer strict operators (===, !==) to avoid unexpected true/false due to type coercion.

4. Logical Operators

Logical operators are used to combine or invert boolean conditions.
Common operators are: AND (&&), OR (||), and NOT (!).
They always return a boolean value (true or false) unless used with non-boolean operands (JS allows truthy/falsy evaluation).

Logical AND (&&)

Returns true only if both operands are true.
# Important: If first operand is false, JS does not evaluate the second (short-circuit). 


console.log(true && false); 
/*
  # result: false
  # reason: both must be true → second operand ignored if first is false (short-circuit)
*/
console.log(true && true); 
/*
  # result: true
  # reason: both operands are true → AND returns true
*/

Logical OR (||)

Returns true if at least one operand is true.
# Important: If first operand is true, JS does not evaluate the second (short-circuit). 


console.log(true || false); 
/*
  # result: true
  # reason: at least one operand is true → OR returns true
  # short-circuit: second operand ignored if first is true
*/
console.log(false || false); 
/*
  # result: false
  # reason: both operands are false → OR returns false
*/

Logical NOT (!)

Inverts the boolean value of an operand.
# Important: Converts truthy/falsy values to boolean first, then inverts. 


console.log(!true); 
/*
  # result: false
  # reason: NOT inverts true → false
*/
console.log(!false); 
/*
  # result: true
  # reason: NOT inverts false → true
*/
console.log(!0); 
/*
  # result: true
  # reason: 0 is falsy → NOT converts to true
*/
console.log(!"Hello"); 
/*
  # result: false
  # reason: non-empty string is truthy → NOT converts to false
*/

# Additional info: Logical operators are commonly used in conditional statements.
# They can also return the actual value (not just boolean) when used with non-boolean values → reason: JS evaluates truthy/falsy and short-circuits.

5. Increment and Decrement Operators

Increment and decrement operators increase or decrease a variable’s value by 1.
They can be used in two forms: pre (e.g., ++x, --x) and post (e.g., x++, x--).
# Important: The position (pre or post) determines whether the value is changed before or after evaluation.

Post-Increment (x++)

Increases the variable by 1, but returns the value before incrementing.
# Important: The increment happens after the current value is used. 


let y = 5;
console.log(y++); 
/*
  # result: 5
  # reason: returns current value first → then increments y to 6
*/
console.log(y); 
/*
  # result: 6
  # reason: value updated after previous operation
*/

Pre-Increment (++x)

Increases the variable by 1, and returns the value after incrementing.
# Important: The increment happens before the current value is used. 


let z = 5;
console.log(++z); 
/*
  # result: 6
  # reason: increments first → then returns updated value
*/

Post-Decrement (x--)

Decreases the variable by 1, but returns the value before decrementing.
# Important: The decrement happens after the current value is used. 


let a = 5;
console.log(a--); 
/*
  # result: 5
  # reason: returns current value first → then decrements a to 4
*/
console.log(a); 
/*
  # result: 4
  # reason: value updated after previous operation
*/

Pre-Decrement (--x)

Decreases the variable by 1, and returns the value after decrementing.
# Important: The decrement happens before the current value is used. 


let b = 5;
console.log(--b); 
/*
  # result: 4
  # reason: decrements first → then returns updated value
*/

# Additional info: Increment and decrement operators are commonly used in loops and iterative calculations.
# Pre vs Post determines whether the current operation uses the old value or the updated value → reason: affects expressions where the operator is combined with other operations.

6. Ternary Operator

The ternary operator is a shorthand for if...else statements.
Syntax: condition ? valueIfTrue : valueIfFalse
# Important: Useful for concise conditional assignments.

Basic Usage


let age = 20;
let result = (age >= 18) ? "Adult" : "Minor";
console.log(result); 
/*
  # result: "Adult"
  # reason: age is 20 → condition (age >= 18) is true → valueIfTrue is returned
  # note: concise replacement for multi-line if...else statement
*/

Real-World Application Example

Imagine a website showing a discount based on user membership.
The ternary operator can determine the discount quickly without using if...else. 


let membershipType = "Premium";
let discount = (membershipType === "Premium") ? 30 : 10;
console.log(`Discount: ${discount}%`);
/*
  # result: "Discount: 30%"
  # reason: user is Premium → condition true → 30% discount applied
  # alternate case: if membershipType was "Basic", discount would be 10%
  # note: ternary operator makes conditional assignment concise and readable
*/

# Additional info: Ternary operator can be nested for multiple conditions,
# but readability may decrease → reason: only use nested ternary for simple conditions.

7. Bitwise Operators

These operators work at the binary (bit) level: & (AND), | (OR), ^ (XOR), ~ (NOT), << (left shift), >> (right shift). 


console.log(5 & 3);  // 1  (0101 & 0011 = 0001)
console.log(5 | 3);  // 7  (0101 | 0011 = 0111)
console.log(5 ^ 3);  // 6  (0101 ^ 0011 = 0110)
console.log(~5);     // -6 (bitwise NOT)
console.log(5 << 1); // 10 (shift left)
console.log(5 >> 1); // 2  (shift right)

8. Nullish Coalescing (??)

The nullish coalescing operator ?? returns the right-hand value only if the left-hand value is null or undefined. 


let name = null;
console.log(name ?? "Guest"); // Guest

9. Optional Chaining (?.)

Optional chaining prevents errors when accessing nested object properties that might not exist. Instead of crashing, it returns undefined. 


let user = { profile: { name: "Alex" } };
console.log(user.profile?.name);   // Alex
console.log(user.contact?.email);  // undefined (safe, no error)

10. Additional JavaScript Operator Topics

To become proficient with JavaScript operators, it’s important to understand additional behaviors, type conversions, and less common operators.
# These topics go beyond basic arithmetic, comparison, logical, and ternary operators.

1. String Concatenation with +

The + operator can be used to concatenate strings.
# Important: If any operand is a string, JS converts the other operand to string and concatenates. 


let text = "Hello, " + "World!";
console.log(text); 
/*
  # result: "Hello, World!"
  # reason: both are strings → + concatenates
*/

let value = "Age: " + 25;
console.log(value); 
/*
  # result: "Age: 25"
  # reason: number 25 converted to string → concatenated with "Age: "
*/

2. Unary + and -

Unary + converts a value to a number.
Unary - converts to number and negates it.
# Important: Useful for converting strings to numbers. 


let strNum = "42";
console.log(+strNum); 
/*
  # result: 42 (number)
  # reason: unary + converts string to number
*/
console.log(-strNum); 
/*
  # result: -42
  # reason: unary - converts string to number and negates it
*/

3. Comma Operator (,)

Evaluates multiple expressions and returns the value of the last one.
# Important: Useful in for loops or concise assignments. 


let a = (1, 2, 3);
console.log(a); 
/*
  # result: 3
  # reason: comma operator evaluates 1, 2, then 3 → returns last value
*/

4. Exponentiation Operator (**)

Performs power calculation: a ** b → a raised to the power b.
# Important: Can also be combined with assignment: **= 


let x = 2;
x **= 3; 
console.log(x); 
/*
  # result: 8
  # reason: 2 ** 3 → 2*2*2 = 8, assigned back to x
*/

5. typeof Operator

Returns the type of a variable or value.
# Important: Useful for type checking and debugging. 


console.log(typeof 42); 
/*
  # result: "number"
*/
console.log(typeof "Hello"); 
/*
  # result: "string"
*/
console.log(typeof null); 
/*
  # result: "object" → historic quirk in JS
*/

6. void Operator

Evaluates an expression and returns undefined.
# Important: Mostly used to prevent return values from expressions (e.g., links in HTML). 


console.log(void 0); 
/*
  # result: undefined
  # reason: void always returns undefined regardless of the expression
*/

7. Operator Precedence & Associativity

Determines the order in which operators are evaluated.
# Important: Misunderstanding can lead to unexpected results. 


console.log(2 + 3 * 4); 
/*
  # result: 14
  # reason: * has higher precedence than + → 3*4=12 → 2+12=14
*/

console.log((2 + 3) * 4); 
/*
  # result: 20
  # reason: parentheses override precedence → 2+3=5 → 5*4=20
*/

# Additional info: Understanding type coercion, truthy/falsy values, and operator precedence is key to mastering JavaScript operators.
# Practice combining operators in expressions to become fluent.

11. JavaScript Operator Precedence Table

Operator precedence determines the order in which operations are evaluated.
# Important: Higher precedence operators are evaluated before lower ones.
# Associativity determines the direction of evaluation when operators have the same precedence.

new Create new object 20 Right → Left
++x, --x, +, -, ~, ! Unary operators (increment, decrement, unary plus/minus, bitwise NOT, logical NOT) 19 Right → Left
* / % Multiplication, Division, Modulus 18 Left → Right
+ - Addition, Subtraction (also string concatenation with +) 17 Left → Right
<< >> >>> Bitwise shift operators 16 Left → Right
< <= > >= Relational comparison 15 Left → Right
== != === !== Equality / Inequality 14 Left → Right
& Bitwise AND 13 Left → Right
^ Bitwise XOR 12 Left → Right
| Bitwise OR 11 Left → Right
&& Logical AND 10 Left → Right
|| Logical OR 9 Left → Right
? : Ternary conditional 8 Right → Left
= += -= *= /= %= **= Assignment operators 7 Right → Left
, Comma operator 6 Left → Right

# Additional info: Understanding operator precedence avoids unexpected results in complex expressions.
# Tip: Use parentheses () to make order explicit and improve readability.
